## ❌ 错题1：create_task 后不 await 的异常处理

**题目**: `create_task` 创建 Task 后，如果后面一直不 `await` 它，异常会被吞掉吗？

**正确答案**:
- Task 确实会在后台跑（已经排入事件循环）
- 但如果 Task 内部抛异常，而你又没 `await` 它 → **异常被吞掉**
- Python 会打印警告 `Task exception was never retrieved`，然后静默消失

**铁律**：永远 `await` 你的 Task！

```python
task = asyncio.create_task(fetch(url))
# ...
result = await task   # 必须 await，否则异常被吞
```

## ❌ 错题2：create_task + 列表推导 vs gather 的异常行为

**题目**: 下面两段代码，异常处理行为一样吗？

```python
# 代码 A
tasks = [asyncio.create_task(fetch(u)) for u in urls]
results = [await t for t in tasks]

# 代码 B
results = await asyncio.gather(*[fetch(u) for u in urls])
```

**正确答案**:
- **默认行为一样**：都是"一个崩全部崩"（抛异常）
- 代码 A 的隐患更大：列表推导中断后，后面的 task 没人 `await` 了 → **异常被吞**
- 代码 B 可用 `return_exceptions=True` 防御，代码 A 不行

**正确做法**：批量并发用 `gather(..., return_exceptions=True)`

```python
results = await asyncio.gather(
    *[fetch(u) for u in urls],
    return_exceptions=True   # 一个失败不影响其他
)
```

## ❌ 错题3：Future 只能设置一次结果

**题目**: Future 被 `set_result("ok")` 后，还能再 `set_result("new")` 吗？

**正确答案**:
- **不能！** Future 是"一次性契约"
- `set_result` / `set_exception` / `cancel` 三选一，只能调用一次
- 第二次调用会抛 `InvalidStateError`

```python
f = asyncio.Future()
f.set_result("ok")
# f.set_result("new")  # ❌ InvalidStateError!
```

**和变量赋值的区别**：变量可以反复赋值，Future 不行。

## ❌ 错题4：Semaphore + gather 综合应用

**题目**: 写 `fetch_all(urls)`，要求最多 3 个并发，一个失败不影响其他，返回所有结果。

**正确答案**:

```python
import asyncio
from typing import List

async def fetch_all(urls: List[str], max_concurrency: int = 3):
    semaphore = asyncio.Semaphore(max_concurrency)
    
    async def limited_fetch(url):
        async with semaphore:      # 每个请求单独拿通行证
            return await fetch(url)
    
    return await asyncio.gather(
        *[limited_fetch(u) for u in urls],
        return_exceptions=True      # 一个崩了不影响其他
    )
```

**关键点**:
- `semaphore` 包在内部函数里 = 每个 URL 请求时都限流
- `gather(..., return_exceptions=True)` = 错误变成 Exception 对象混在结果里

## ❌ 错题5：run_in_executor 用法

**题目**: FastAPI 路由里调用同步第三方库（如 `requests.get`），正确做法是什么？

**正确答案**:

```python
import asyncio
import requests

@router.get("/external")
async def call_external():
    loop = asyncio.get_running_loop()
    
    # ✅ 丢进线程池，事件循环不卡死
    result = await loop.run_in_executor(
        None,               # None = 默认 ThreadPoolExecutor
        requests.get,       # 同步函数
        "https://api.xxx.com"  # 参数
    )
    return result.json()
```

**什么时候不用 run_in_executor？**
- 如果用的是 `httpx` 等异步库 → 直接 `await httpx.get()`，不需要 `run_in_executor`

**ThreadPoolExecutor vs ProcessPoolExecutor**:
- `None`（默认 ThreadPoolExecutor）→ 适合 IO 密集型（网络、文件）
- `ProcessPoolExecutor()` → 适合 CPU 密集型（大循环、图片处理），真·多核并行
