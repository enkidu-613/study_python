# 21. GitHub 注册、安全设置与个人主页
> **一句话理解：GitHub 是存放远程仓库和协作代码的平台；Git 是本机记录历史的工具。**
## 学什么，不学什么
学：注册账号、安全设置、公开和私有仓库。 不学：上传代码；下一章做。
## 术语
GitHub account=平台账号；public=所有人可见；private=授权成员可见；2FA=第二重登录验证。
## 最小模板
访问 [GitHub](https://github.com)，选择 **Sign up**，用自己的邮箱完成验证。登录后进入头像菜单的 **Settings**：设置强密码，并按页面提示开启双重验证或通行密钥。回到个人主页，确认能看到自己的用户名。
## 常见坑
- 把 GitHub 密码或验证码写进代码、聊天或 README。
- 误以为注册 GitHub 就已经把本地代码上传。
- 创建 public 仓库前没检查是否含 `.env`、密码或私人文件。
## 检查点
- [ ] 能登录 GitHub 并打开自己的个人主页。
- [ ] 知道 public 与 private 的可见范围。
## 小练习
在个人资料 Bio 写一句非敏感的学习目标；保存后刷新确认显示。
