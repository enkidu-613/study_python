# 00. 从零到 AI 应用开发的学习地图

> **一句话理解：AI 应用开发工程师像产品工程师——把已有模型、业务数据和软件功能组合成用户真正能用的服务。**

## 本章学什么，不学什么

学：这条路线的终点、每一阶段的作用，以及“用模型”和“训练模型”的边界。

不学：任何 Python 语法、模型原理或框架名称。那些从下一章开始逐步出现。

## 准确术语

| 术语 | 准确含义 |
| --- | --- |
| AI 应用 | 用户输入后，软件调用模型并返回结果的产品功能。 |
| 预训练模型 | 已经由模型提供方训练好的模型，你通过本地加载或网络接口使用它。 |
| API | 一个程序按约定向另一个程序发请求、拿结果的接口。 |
| 后端 | 接收请求、处理业务规则、访问数据与调用模型的程序。 |
| 前端 | 用户点击、输入和看到结果的界面。 |

## 最小模板：识别一个 AI 应用的组成

```text
用户提问
  → 网页或手机界面
  → Python 后端
  → 检索业务资料 / 调用模型 API
  → 返回可阅读的回答
```

例如：课程资料问答助手、客服草稿助手、文件总结工具、带工具调用的工作流。现在只需要能读懂这张图，不需要运行任何代码。

## 三件容易混淆的事

| 做法 | 做了什么 | 适合什么 |
| --- | --- | --- |
| Prompt | 本次请求临时交代要求 | 语气、格式、规则 |
| RAG | 本次请求临时找资料再交给模型 | 会变化的公司文档、可引用知识 |
| 微调 | 用示例长期改变模型的一部分行为 | 稳定分类或固定输出习惯 |

本路线重点是前两项，也会学会判断第三项何时才值得考虑。

## 常见坑

- 以为“学习 AI”必须先学高等数学：做 AI 应用不需要先走训练路线。
- 只会调用接口就急着做产品：没有 Python、网络和调试基础，报错时会卡住。
- 把 Agent 当魔法：它仍是代码、模型和工具调用组成的可检查流程。

## 检查点

- [ ] 能说出 AI 应用工程师的目标是“交付产品”，不是“从零训练模型”。
- [ ] 能用一句话区分 Prompt、RAG 和微调。
- [ ] 知道下一章先学电脑文件与开发环境。
