# 02. 终端命令与程序运行

> **一句话理解：终端是用文字精确指挥电脑的窗口；命令总是在“当前目录”里执行。**

## 本章学什么，不学什么

学：打开终端、查看当前位置、进入目录、列出文件、运行 Python、看第一条报错。

不学：复杂 Shell 脚本、服务器运维或危险删除命令。

## 准确术语

| 术语 | 准确含义 |
| --- | --- |
| Terminal/终端 | 输入命令并显示程序输出的应用。 |
| Shell | 读取命令并启动程序的命令解释器。 |
| 当前目录 | 此刻命令默认操作的文件夹。 |
| 命令 | 让 Shell 执行的文本，例如 `python3 hello.py`。 |
| 退出码 | 程序结束时交给系统的结果；`0` 通常表示成功。 |

## 最小模板：定位并运行程序

先打开适合自己系统的终端：Windows 按开始菜单并搜索 **PowerShell**；macOS 在“应用程序 → 实用工具”打开 **Terminal**；Linux 从应用菜单打开 **Terminal**。

将路径替换成你自己的项目目录，再只执行对应一列：

| 目标     | Windows PowerShell               | macOS / Linux Shell          |
| ------ | -------------------------------- | ---------------------------- |
| 显示当前目录 | `Get-Location`                   | `pwd`                        |
| 进入项目目录 | `cd $HOME\Documents\ai-beginner` | `cd ~/Documents/ai-beginner` |
| 列出文件   | `Get-ChildItem`                  | `ls`                         |
| 运行程序   | `python hello.py`                | `python3 hello.py`           |

PowerShell 中 `cd` 是 `Set-Location` 的简写；macOS/Linux 中 `cd` 是 Shell 内建命令。它们都改变“当前目录”。

## 如何读最小报错

故意把 `print` 改成 `prnit` 后运行：

```text
NameError: name 'prnit' is not defined
```

先读最后一行：它说明 Python 找不到叫 `prnit` 的名字。再看上方的文件名和行号，回到对应位置修正。

## 常见坑

- 在错误目录运行：先用 `pwd` 和 `ls` 确认。
- Windows 用户在 PowerShell 中应使用 `Get-Location` 和 `Get-ChildItem`；它们分别对应 macOS/Linux 的 `pwd` 和 `ls`。
- 文件名有空格却不加引号：新手阶段先使用英文、小写和下划线文件名。
- 看到英文报错就全部复制到搜索框：先看最后一行、文件名和行号。

## 检查点

- [ ] 能用自己系统的命令判断当前目录。
- [ ] 能用 `cd` 进入项目目录，并列出 `hello.py`。
- [ ] Windows 能用 `python hello.py`、macOS/Linux 能用 `python3 hello.py` 运行程序。
- [ ] 知道报错先看最后一行和行号。
