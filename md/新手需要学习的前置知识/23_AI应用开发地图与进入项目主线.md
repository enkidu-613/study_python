# 23. AI 应用开发地图与进入项目主线
> **一句话理解：AI 应用工程师用代码约束输入、补充资料、调用已有模型并交付可靠服务。**
## 学什么，不学什么
学：API、Prompt、RAG、Agent、微调的边界。 不学：训练模型。
## 术语
Prompt=本次指令；RAG=本次检索资料后回答；Agent=模型在受控代码中调用工具；微调=用示例长期改变模型行为。
## 最小模板
```text
用户输入 → 后端校验 → 可选检索资料 → 模型 API → 输出校验 → 返回用户
```
模型不是权限系统；工具、输入、输出和日志都要由后端代码约束。
## 常见坑
- 把会变化的公司资料塞进微调，通常应优先 RAG。
- 让 Agent 无限制调用工具。
- 只看回答流畅，不评估正确性和来源。
## 检查点
- [ ] 能区分 Prompt、RAG、Agent、微调。
- [ ] 知道模型训练不属于当前路线。

为“课程资料问答”写六步数据流。
