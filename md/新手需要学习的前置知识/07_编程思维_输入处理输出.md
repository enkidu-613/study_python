# 07. 编程思维：输入、处理、输出

> **一句话理解：程序把明确输入按明确规则处理，再给出可验证输出。**

## 学什么，不学什么

学：IPO（输入、处理、输出）拆需求。  
不学：复杂算法。

## 术语

| 输入 | 处理 | 输出 |
| --- | --- | --- |
| 用户给程序的数据 | 程序按规则做的步骤 | 程序给用户的结果 |

## 最小模板

需求“判断分数是否及格”：输入=分数；处理=比较是否≥60；输出=及格或复习。写代码前先写这三行。

```python
score = 80
print(score >= 60)
```

## 常见坑

- 没定义输入就开始写代码。
- 没写预期输出，无法判断正确。
- 一次解决十件事；先完成一个最小流程。

## 检查点

- [ ] 能用 IPO 描述“温度换算”。

## 小练习

为“待办计数器”写 IPO，不写代码。
