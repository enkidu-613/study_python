# 10. 数据结构与性能直觉

> **一句话理解：数据结构是组织数据的方式；选对 list、dict、set，代码会更清晰，也常常更快。**

## 本章学什么，不学什么

学：list、dict、set 的选择，以及线性查找和哈希查找的直觉。

不学：刷算法题、复杂排序、树、图或面试技巧。

## 准确术语

| 术语 | 准确含义 |
| --- | --- |
| 数据结构 | 数据在程序中的组织方式及其常用操作。 |
| list | 有顺序、允许重复，适合按顺序遍历。 |
| dict | 用唯一键快速找到对应值的映射。 |
| set | 不重复值的集合，适合判断“是否出现过”。 |
| 查找 | 从一组数据里找到目标项目。 |
| 线性查找 | 从头到尾一个个比较。 |
| 哈希查找 | 通过键或值的哈希定位；平均情况下通常很快。 |

## 最小模板：按需求选择容器

```python
task_list = ["学习 Python", "写笔记", "学习 Python"]
user_by_id = {
    101: "小明",
    102: "小红",
}
completed_titles = {"学习 Python", "喝水"}

print(task_list[0])
print(user_by_id[101])
print("喝水" in completed_titles)
```

`task_list[0]` 通过位置取第一个任务；`user_by_id[101]` 通过 ID 取用户；`"喝水" in completed_titles` 判断是否存在。三种写法相近，但用途不同。

## 性能直觉：别先背公式

如果有 10,000 条用户记录：

```python
users = [
    {"id": 101, "name": "小明"},
    # ... 更多用户
]
```

为了找到 `id == 9999`，list 可能从前往后看很多条。若把它组织成：

```python
user_by_id = {
    101: {"id": 101, "name": "小明"},
    # ... 更多用户
}
```

就可以直接用 `user_by_id[9999]`。直觉上：**经常按唯一编号找东西，就用 dict；经常只问“有没有”，就用 set；要保留顺序和重复项，就用 list。**

## 常见坑

- 为了“快”到处改 dict：如果需要顺序与重复值，list 更准确。
- 用 list 判断大量项目是否存在：重复检查时通常更适合 set。
- 以为 dict 永远不会出错：键不存在时 `data["missing"]` 会报错；可用 `data.get("missing")`。
- 把本章当算法面试课：目标是写应用时做合适选择。
