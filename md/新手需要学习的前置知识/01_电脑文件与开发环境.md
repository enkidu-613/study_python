# 01. 电脑文件与开发环境

> **一句话理解：写程序就是把文本保存成特定文件，再交给正确的程序执行。**

## 本章学什么，不学什么

学：文件、文件夹、路径、扩展名，以及编辑器和 Python 解释器的分工。

不学：终端命令；下一章才用命令操作文件。

## 准确术语

| 术语 | 准确含义 |
| --- | --- |
| 文件 | 保存内容的单位，例如 `hello.py`。 |
| 文件夹/目录 | 用来装文件和其他文件夹的容器。 |
| 路径 | 文件所在位置；绝对路径从磁盘根位置开始，相对路径从当前目录开始。 |
| 扩展名 | 文件名最后的类型提示；Python 源代码通常是 `.py`。 |
| 编辑器 | 用来写代码的应用，例如 PyCharm 或 VS Code；它不负责执行 Python。 |
| Python 解释器 | 读取并执行 `.py` 文件的程序。 |

## 最小模板：建立第一个项目文件夹

在“文稿”中新建文件夹：

```text
ai-beginner/
└── hello.py
```

把下面内容保存为 `hello.py`。注意：它是普通文本，不是 Word 文档。

```python
print("你好，AI 应用开发！")
```

`print` 是 Python 的内置函数；Python 执行它时，会把括号里的文本显示在屏幕上。

## 路径例子

```text
/Users/你的用户名/Documents/ai-beginner/hello.py   ← 绝对路径示例
ai-beginner/hello.py                               ← 相对路径示例
```

不要死记自己的完整路径；关键是理解“相对路径必须有一个当前起点”。

## 常见坑

- 文件实际叫 `hello.py.txt`：系统隐藏扩展名时很常见。
- 把代码写进富文本编辑器：会混入不可见格式。
- 以为安装 PyCharm 就安装了 Python：编辑器和解释器是两个东西。
