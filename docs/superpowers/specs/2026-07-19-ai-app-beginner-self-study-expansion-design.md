# AI 应用开发新手自学课程扩写设计

## 目标

将 `md/新手需要学习的前置知识/` 从 13 篇概览型文档重构为 24 篇可独立完成的小章节，使完全没有编程经验的朋友可以仅依靠文字、可复制命令和检查点完成自学。

## 呈现原则

- 使用纯文字、代码块、表格和检查点；不依赖截图或视频。
- 每章只完成一个可验证动作，避免在一篇文档塞进多个新概念。
- 命令均提供 Windows PowerShell 与 macOS/Linux Shell 的对应写法。
- 每章使用一致的教学合同：一句话理解、范围、术语、最小模板、常见坑、检查点、小练习和下一步。
- 练习采用读懂、跟做、独立改写的顺序；不以“看完”为完成标准。

## Git 与 GitHub 的完整最小链路

Git 内容拆成 `15` 至 `19` 五章，按不可跳过的实际顺序教学：

1. 安装 Git、运行 `git --version`、配置 `user.name` 与 `user.email`。
2. 初始化本地仓库、检查状态、暂存、提交、查看差异和恢复未提交修改。
3. 注册 GitHub、设置强密码与双重验证、理解公开/私有仓库和个人资料。
4. 在 GitHub 网页创建空仓库；在本地配置 `origin`，执行首次 `push`，在网页验证文件存在。
5. `clone`、`pull`、修改后 `push`；以最小范围理解分支和 Pull Request，不要求团队协作。

不教令牌创建、SSH 配置、冲突解决或高级分支策略；遇到需要认证时解释“浏览器授权或令牌由 GitHub 管理”，并让学习者使用官方界面完成，而不是在文档中放置敏感凭据。

## 章节层次与交接

`01` 至 `06` 解决“能建立并运行程序”；`07` 至 `14` 解决“能独立写、组织、调试 Python 小项目”；`15` 至 `19` 解决“能安全保存并上传代码”；`20` 至 `22` 解决“能读懂后续 API、数据库和数据选择”；`23` 只解释 AI 应用部件边界，并链接到既有 `md/00_环境配置与PyCharm使用.md`。

## 验收标准

- 目录含 README 与 `00` 至 `23` 共 25 个 Markdown 文件。
- GitHub 账号注册、账号安全、创建远程仓库、首次推送和网页验证均有独立、可执行步骤。
- 所有系统相关步骤均提供 Windows PowerShell 与 macOS/Linux Shell 的对应信息。
- 安装、授权和账号页面使用文字定位与“应看到什么”的检查点，不依赖截图。
- 不修改既有项目的 `md/00_环境配置与PyCharm使用.md` 至 `md/32_HuggingFace生态.md`。
- 不进入模型训练、算法岗或需要真实密钥的内容。
