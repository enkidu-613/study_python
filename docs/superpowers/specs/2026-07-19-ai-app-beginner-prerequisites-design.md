# AI 应用开发新手前置课程设计

## 目标

在 `md/新手需要学习的前置知识/` 创建一套从零开始、独立可读的 Markdown 课程。学习者完成这套课程后，具备开始本项目 AI 应用开发主线所需的编程与工程基础，而不是模型训练或算法岗位能力。

## 目标学习者

- 会使用电脑和浏览器，但不了解编程、命令行、网络或数据库。
- 目标是尽快独立开发可运行的 AI 应用，而不是从零训练模型。
- 可以使用 macOS；课程中的核心概念需可迁移到 Windows/Linux。

## 范围与边界

课程覆盖文件管理、终端、编程思维、Python、Git、HTTP/JSON、数据库/SQL、轻量数据结构与 AI 应用概念地图。每个主题以能运行、能验证和能继续推进项目为完成标准。

课程不覆盖高等数学推导、经典机器学习算法、深度学习训练、反向传播、PyTorch 训练循环、LoRA/QLoRA 实操、CUDA 或分布式训练。微调只在最后一章作为“什么时候选择它”的概念边界出现。

## 文件结构

```text
md/新手需要学习的前置知识/
├── README.md
├── 00_从零到AI应用开发的学习地图.md
├── 01_电脑文件与开发环境.md
├── 02_终端命令与程序运行.md
├── 03_编程思维与问题拆解.md
├── 04_Python入门_变量条件循环.md
├── 05_Python核心_函数容器与文件.md
├── 06_Python项目化_模块异常调试与依赖.md
├── 07_Git与GitHub_保存代码历史.md
├── 08_互联网_HTTP与JSON.md
├── 09_数据库与SQL入门.md
├── 10_数据结构与性能直觉.md

```

`README.md` 提供阅读顺序、总时长建议、每章产出与进入现有项目的入口。`00` 说明路线、学习方式和“AI 应用开发不等于模型训练”的职业边界。`01` 至 `10` 按依赖顺序提供独立的前置基础。`11` 把学习者交接到本项目现有 `md/00_环境配置与PyCharm使用.md` 及后续 AI 应用主线。

## 每章教学合同

除 README 外，每章使用一致、紧凑的结构：

1. 一句话理解。
2. 本章学什么与不学什么。
3. 准确术语表。
4. 最小可运行代码或命令。
5. 两到四个常见坑。
6. 可立即执行的检查点。
7. 一个小练习与下一章入口。

首次出现的执行对象必须说明其代码形态、从哪里创建或导入、输入、输出及调用者。练习只依赖同章和此前章节的内容，避免隐式前置知识。

## 章节职责

| 章节 | 学习者完成后能做什么 |
| --- | --- |
| 01 | 管理文件、识别路径、安装开发工具并理解解释器和编辑器的分工。 |
| 02 | 在终端定位目录、创建/运行 Python 文件、读懂最基本的退出与报错信息。 |
| 03 | 将自然语言需求拆为输入、处理、输出和可验证的小步骤。 |
| 04 | 写出变量、条件、循环与最小 Python 程序。 |
| 05 | 用函数、list、dict 和文件读写完成小型数据处理。 |
| 06 | 用模块、虚拟环境、依赖、异常和调试方式维护一个小 Python 项目。 |
| 07 | 用 Git 保存、比较和恢复代码历史，并避免提交密钥。 |
| 08 | 解释浏览器/客户端、服务器、URL、HTTP、JSON 和状态码的数据流。 |
| 09 | 设计一张简单表并用 SQL 完成增删改查。 |
| 10 | 根据用途选择 list、dict、set，并理解线性扫描与哈希查找的性能差异。 |
| 11 | 区分 Prompt、RAG、Agent、模型 API、微调和模型训练，并进入项目主线。 |

## 验收标准

- 目录内有 README 和 `00` 至 `11` 共 13 个 Markdown 文件。
- 每章均可脱离旧教程理解，不依赖读者已经学过编程。
- 每章具有明确的边界、最小模板、常见坑和检查点。
- 没有训练模型或算法岗的教学任务。
- README 和 `11` 都提供进入现有项目主线的具体文件链接。
- 不修改现有 `md/00_环境配置与PyCharm使用.md` 至 `md/32_HuggingFace生态.md` 的内容。
