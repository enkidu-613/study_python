## 对话 71: pytest 单元测试实战踩坑与 pyproject.toml 深度理解

**时间**: 2026-06-23 04:31 (NTP 同步)

### 内容

1. **pyproject.toml 项目配置**：解释 pyproject.toml 作为 Python 项目单一事实来源（Single Source of Truth），管理项目名称/版本、生产依赖、开发依赖、Poetry 配置、工具设置（pytest）的所有区块作用。
2. **pytest 模块路径错误**：`ModuleNotFoundError: No module named 'main'` 的根本原因是 test/ 目录运行时不在 Python 模块搜索路径；方案2（`pyproject.toml` 配置 `[tool.pytest.ini_options] pythonpath = ["."]`）vs 方案3（`test/__init__.py`）的对比，以及现代 pytest 不建议加 `test/__init__.py` 的原因（pytest 6.0+ 行为变化 + PEP 420 隐式命名空间包）。
3. **routers/__init__.py 的必要性**：`__init__.py` 在应用包（如 `routers/`）和测试包（`test/`）中的不同角色——前者是应用架构的核心包聚合，后者是现代不推荐的花瓶标记文件。
4. **os 模块与环境变量**：`import os; os.getenv("DATABASE_URL")` 的用途，os 作为 Python 操作系统万能工具箱（环境变量、文件路径、进程等）的边界介绍。
5. **SQLite `check_same_thread=False`**：SQLite 的线程安全检查机制，在 FastAPI 异步多线程场景下必须关闭，否则会抛出 `SQLite objects created in a thread can only be used in that same thread` 错误。
6. **pytest conftest.py 导入缺失**：`NameError: name 'get_db' is not defined` — conftest.py 中使用了 `get_db` 但没有导入，添加 `from database import get_db` 修复。
7. **pytest fixture 参数注入**：`NameError: name 'client' is not defined` — 测试函数必须把 fixture 名作为参数写进函数签名，pytest 才会自动注入。
8. **APIRouter 对象没有 get_db 属性**：`AttributeError: 'APIRouter' object has no attribute 'get_db'` — `get_db` 是通过 Depends 在路由函数参数中注入的，不是 router 对象的属性，相关单元测试思路有误。
9. **auth 路由测试通过**：`test_login_then_get_current_user` 功能测试 2 项全部通过，验证了注册→登录→获取当前用户的完整认证流程。

## 对话 72: pytest 生命周期、测试库隔离与流式接口测试

**时间**: 2026-06-23 04:56

### 内容

1. **接口测试分层**：明确工作中常见的 Router、Service、Repository、Database 分层，以及单元测试、Repository 集成测试、API 测试和少量 E2E 测试的职责边界。
2. **TestClient 与 app**：确认 `app` 是 `main.py` 中的 FastAPI 应用实例；`TestClient(app)` 无需启动 Uvicorn 即可直接请求应用。
3. **测试目录与首个接口测试**：确认 `test/` 可以使用，但行业更常见 `tests/`；`/docs` 与 Todo 创建测试通过，并识别 `extra_body` 是非阻塞警告。
4. **测试数据库隔离**：逐行理解 `Base.metadata.create_all()`、`test_engine`、`TestingSessionLocal`、`override_get_db`、`dependency_overrides`、`drop_all()` 的作用和执行顺序。
5. **fixture 生命周期**：澄清 `@pytest.fixture` 是 pytest 的依赖注入登记机制，不是中间件；测试参数触发 fixture，`yield` 前准备、测试执行、`yield` 后清理。
6. **with 与异步生命周期**：区分 `with`、`while`、`async def`、`@asynccontextmanager`；`with` 通过 `__enter__/__exit__` 管理 TestClient，进入时启动 lifespan，离开时关闭资源。
7. **fixture 与 Depends 对比**：两者都是依赖注入思想，但 fixture 运行在测试生命周期，`Depends` 运行在 HTTP 请求生命周期。
8. **同名 get_db 修复**：发现 Todo 路由的局部 `get_db` 与 `database.get_db` 不是同一函数对象，导致测试表面通过但实际写入 `my_database.db`；保留旧逻辑注释并改用公共依赖，验证测试前后正式 Todo 数量均为 3。
9. **模块与 APIRouter**：解释 `import_module("routers.todos_routers")` 返回整个模块，`APIRouter` 只是模块内的路由注册对象；`as` 是本地别名，`is` 用于检查对象身份。
10. **第八关范围与节奏调整**：确定先测试 WebSocket 固定收发，再用 monkeypatch 测一个 SSE 响应的状态码、`text/event-stream` 和 `data: ...\n\n` 格式；不测试真实模型原文、token 到达速度、背压和重连。用户反馈本章学习偏慢，后续完成这两个最小闭环后尽快结束 pytest 章节。
