## dialog-67: Alembic、ORM、Prisma、Django/FastAPI 边界澄清

- 阶段: alembic
- 类型: dialog
- 主题: Alembic, ORM, Prisma, Django, FastAPI, SQLAlchemy, SQLModel, Base.metadata, Python生态
- 摘要:
  - 用户把 Alembic 看成类似 ORM，进一步联想到 JavaScript 生态里的 Prisma，因为 Prisma 同时包含 ORM Client、schema 管理和 migration 能力。
  - 澄清：Alembic 不是 ORM，它对应 Prisma Migrate；SQLAlchemy ORM 才对应 Prisma Client。
  - 总结 Python 生态差异：Django ORM + migrations 更像 Prisma 的一体化体验；FastAPI 项目最主流的专业组合是 SQLAlchemy + Alembic；SQLModel 更贴近 FastAPI，但迁移通常仍配 Alembic。
  - 对比 Django 与 FastAPI：二者都是 Python Web 框架，但 Django 是全家桶 Web 框架，内置 ORM、Admin、migrations；FastAPI 是 API 优先框架，更适合 AI 后端、前后端分离、WebSocket、异步接口和微服务。
  - 回答生产使用情况：传统网站、后台、内容系统和历史存量里 Django 更多；AI 应用、现代 API、新项目、异步服务中 FastAPI 增长快且常用。
  - 解释 `Base.metadata`：它是当前代码里所有继承同一个 `Base` 的 ORM 表结构总目录，包含表名、字段、类型、主键、外键、约束等；不是数据、不是数据库连接、也不是查询器。
  - 形成当前阶段的核心记忆：ORM 管数据操作，Alembic 管数据库结构版本变化；`target_metadata = Base.metadata` 表示让 Alembic 用 ORM 模型声明的目标结构去对比真实数据库并生成迁移。
