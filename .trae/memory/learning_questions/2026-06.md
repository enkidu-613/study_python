- question: Few-Shot 为什么不是训练，模型为什么还能“记住”示例？
- answer_summary: Few-Shot 示例只是本次 Prompt 的上下文，不会修改模型参数。模型不是永久记住示例，而是当次请求里读到了示例并模仿。下一次请求如果不带这个示例，模型不会自动继承它。
- exact_rule: Few-Shot = 本次 Prompt 里的临时例题；Fine-Tuning = 用训练数据改变模型参数。

### 2026-06-26 03:52 - Prompt Engineering - temperature/top_p 的真实作用

- question: temperature 是决定模型是否严格遵守 Prompt，top_p 是决定从用户问题中提取关键词的准确度吗？
- answer_summary: temperature 和 top_p 都不是专门控制 Prompt 遵守度或关键词提取准确度的参数。它们控制的是模型生成下一个 token 时的采样随机性：temperature 调整概率分布的尖锐程度，top_p 限制候选 token 池。抽取和分类任务用低随机性，是为了让输出更稳定一致。
- exact_rule: temperature/top_p 控制“怎么选词”，不是直接控制“是否听话”或“是否准确”。

### 2026-06-26 04:25 - Python 工程 - from/import 的思考顺序

- id: lq-20260626-0425-from-import-order
- chapter: md/21_Prompt_Engineering进阶.md
- question: `from ... import ...` 先写来源再写类名很难受，因为实际写代码时先想到的是要用的类或方法。
- answer_summary: Python 的 import 语法是“从哪里拿什么”，但人的思考常常是“我要用什么，它在哪里”。写代码时可以先写类名让 IDE 自动导入，或者先用项目搜索定位来源，再补 import。学习时不必强迫自己从库名开始背，应该从当前要用的对象名开始追来源。
- exact_rule: 人脑顺序可以是“我要用什么 -> 它在哪里”；代码语法最后整理成 `from 来源 import 名字`。

### 2026-06-26 04:32 - Python 工程 - load_dotenv 的作用

- id: lq-20260626-0432-load-dotenv
- chapter: md/21_Prompt_Engineering进阶.md
- question: `load_dotenv()` 在代码中的用处是什么？
- answer_summary: `load_dotenv()` 会读取项目里的 `.env` 文件，并把里面的键值对加载进当前 Python 进程的环境变量。这样后面的 `os.getenv("MODELSCOPE_API_KEY")`、`os.getenv("MODEL_NAME")`、`os.getenv("DATABASE_URL")` 才能读到配置。它用于把密钥、模型地址、数据库地址这类配置从代码里分离出去。
- exact_rule: `load_dotenv()` 负责加载 `.env`；`os.getenv()` 负责读取加载后的变量。
- reinforce_prompt: 为什么不能把 API Key 直接硬编码在 Python 文件里？

### 2026-06-26 04:34 - Python 工程 - dotenv 与 os 的交接

### 2026-06-26 05:59 - Python 工程 - exc 与 raise from

- id: lq-20260626-0559-exc-raise-from
- chapter: md/21_Prompt_Engineering进阶.md
- question: `except Exception as exc` 里的 `exc` 是什么，为什么最后要 `raise HTTPException(...) from exc`？
- answer_summary: `exc` 是被 `except` 捕获到的原始异常对象，例如模型调用失败、解析失败或配置错误。`raise ... from exc` 会抛出新的 `HTTPException`，同时保留“它是由原始异常 exc 导致的”这条因果链，方便日志和调试追踪真正原因。
- exact_rule: `as exc` 保存原始异常；`raise new_error from exc` 保留异常因果链。
- reinforce_prompt: 如果不用 `from exc`，前端响应可能一样，但调试信息会少哪一层？
- status: active

### 2026-06-26 06:01 - Python 调试 - traceback

- id: lq-20260626-0601-python-traceback
- chapter: md/21_Prompt_Engineering进阶.md
- question: Python traceback 是什么，是调试用的吗？
- answer_summary: Traceback 是 Python 报错时打印出的调用栈路线图，用来调试。它会从外层调用一路列到真正出错的那一行，最后一行通常是异常类型和错误信息。看 traceback 时优先看最后一行，再往上找项目自己的文件和行号。
- exact_rule: Traceback = 错误调用链；调试时先看最后一行，再看自己项目里的出错行。
