### 2026-06-26 03:32 - Prompt Engineering - Few-Shot 不是训练

- id: lq-20260626-0332-few-shot-not-training
- chapter: md/21_Prompt_Engineering进阶.md
- question: Few-Shot 为什么不是训练，模型为什么还能“记住”示例？
- answer_summary: Few-Shot 示例只是本次 Prompt 的上下文，不会修改模型参数。模型不是永久记住示例，而是当次请求里读到了示例并模仿。下一次请求如果不带这个示例，模型不会自动继承它。
- exact_rule: Few-Shot = 本次 Prompt 里的临时例题；Fine-Tuning = 用训练数据改变模型参数。
- reinforce_prompt: 用一句话区分 Few-Shot 和 Fine-Tuning。
- status: active

### 2026-06-26 03:32 - Prompt Engineering - 任务提取器 vs 任务规划器

- id: lq-20260626-0332-extraction-vs-planning
- chapter: md/21_Prompt_Engineering进阶.md
- question: 任务提取器是不是应该像大模型安排任务一样列 todo？
- answer_summary: 当前接口是任务提取器，不是任务规划器。任务提取器把自然语言变成稳定字段，例如 title、priority、tags；任务规划器才会把目标拆成 todo 列表。二者都能用 LLM，但输出 Schema 和应用位置不同。
- exact_rule: Extraction 问“这句话里有什么”；Planning 问“为了完成目标要做哪些步骤”。
- reinforce_prompt: 给出一个例子，区分任务提取器和任务规划器的输出。
- status: active
