# 2026-06-17 对话历史

## dialog-68：Alembic 实操踩坑与 Prisma 设计哲学对比

- 日期：2026-06-17
- 阶段：alembic
- 类型：dialog
- 主题：Alembic、Poetry、SQLite、数据库迁移、autogenerate、target_metadata、Prisma、stamp head

### 本次学习内容

用户在 Alembic 第19节教学文档之后进入实操，遇到一系列典型踩坑。

#### Poetry 安装与命令混淆

- `poetry install alembic` ❌ — `install` 不接受参数，只读 `pyproject.toml` 已有依赖。
- `poetry add alembic` ✅ — 新增依赖并安装，同时写入 `pyproject.toml` 和 `poetry.lock`。

#### Poetry 2.0+ 的 shell 变更

- `poetry shell` 在 Poetry 2.0 起默认移除，推荐用 `poetry env activate`。
- 但更省事的方式是直接 `poetry run alembic ...`，在虚拟环境里执行命令。

#### alembic init 找不到命令

Alembic 装在 Poetry 虚拟环境里，外部终端找不到。正确方式：

```bash
poetry run alembic init alembic
```

#### env.py 中 target_metadata 被覆盖

`alembic init` 生成的 `env.py` 默认带一行 `target_metadata = None`，这会覆盖用户自己从 `models` 导入的 `Base.metadata`。导致 `--autogenerate` 报错：

```
Can't proceed with --autogenerate option; environment script ... does not provide a MetaData object
```

修复：删掉或注释掉 `target_metadata = None`。

#### --autogenerate 生成空脚本

成功执行 `poetry run alembic revision --autogenerate -m "init tables"` 后，生成的 `upgrade()` 里只有 `pass`。

原因：数据库 `my_database.db` 里已经有表（之前运行过 `Base.metadata.create_all(bind=engine)`）。Alembic 对比"数据库当前状态"和"模型定义"，差异为零。

#### Prisma vs Alembic 设计哲学

| 工具 | 思维 | 已有数据库时的行为 |
|------|------|------------------|
| **Prisma** | 全量快照 | `db pull` 完整拉回 schema，`migrate dev` 生成含所有表的迁移 |
| **Alembic** | 差异对比 | `--autogenerate` 只生成差异；没差异就空脚本 |

Alembic 没有内置的"反向工程已有数据库为完整迁移"命令，这是设计目标差异。

#### 解决方案

开发环境最干净的做法：

```bash
rm my_database.db
rm alembic/versions/a6220f34feae_init_tables.py
poetry run alembic revision --autogenerate -m "init"
poetry run alembic upgrade head
```

生产环境已有数据时，用 `poetry run alembic stamp head` 标记当前数据库为最新版本，从此只管理增量变更。

### 关键代码地图

- `database.py#L1-L21` — SQLite 数据库配置，`my_database.db`
- `models.py#L1-L30` — SQLAlchemy `Base` 和模型定义
- `alembic/env.py#L1-L84` — Alembic 环境配置，`target_metadata = Base.metadata`
- `alembic/versions/a6220f34feae_init_tables.py` — 初始空迁移脚本

### 学习结论

- Alembic 是"差异优先"工具，不是"全量快照"工具。
- `poetry run` 是 Poetry 2.x 下调用虚拟环境命令的标准方式。
- `target_metadata = None` 是 Alembic init 的默认陷阱，几乎每个初学者都会踩。
- 空迁移脚本 = 数据库已是最新状态，不是 bug。
