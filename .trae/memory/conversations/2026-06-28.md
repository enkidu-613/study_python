## dialog-79：Pydantic default_factory、Python 类实例化、字典解包与后续章节确认

### 阶段

- stage_id: `ai-safety`
- 当前状态：第 22 章 AI 安全与伦理准备中。

### 本次对话摘要

- **`default_factory` 在 Field 中的含义**：用于处理可变类型（list、dict、set）的默认值，避免所有实例共享同一个对象。`Field(default_factory=list)` 每次实例化时调用 `list()` 生成新列表，对比 `Field(default=[])` 的共享陷阱。项目中 `tags: list[str] = Field(default_factory=list)` 当 LLM 返回 JSON 缺少 tags 时自动生成空列表。
- **后续 4 章路线确认**：AI 安全与伦理 → RAG Chunking 策略 → RAG 评估与指标 → AI Agents 基础。
- **Python 类实例化**：Python 没有 `new` 关键字，直接 `ClassName(args)` 即可，背后自动调用 `__new__` 分配内存 + `__init__` 初始化。
- **Pydantic BaseModel 实例化**：和普通类一样直接调用，额外附带类型校验。也可以从字典创建（`**` 解包）或 JSON 字符串创建（`model_validate_json`）。
- **`**` 字典解包**：把字典拆成键值对传给函数，`MyModel(**data)` 等价于 `MyModel(key1=val1, key2=val2)`。兄弟符号 `*` 是列表解包。

### 关键结论

- 可变类型默认值一律用 `default_factory`，避免实例间共享导致污染。
- Python 实例化 = 类名加括号，没有 `new`。
- Pydantic BaseModel 实例化语法和普通类一样，多了类型校验。
- `**` 是字典解包，`*` 是列表解包。
