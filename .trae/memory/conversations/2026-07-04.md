### 关键结论

- Function Calling 是能力/机制/流程；tool call 是模型输出里的具体工具调用请求。
- 本章边界：第 25 章会画 Agent Loop；第 26 章开始学习执行 Loop。
- `required` 的准确写法是 `"required": ["query"]`，它和 `properties` 同级，都在 `parameters` 里面。
- `TOOLS` 给模型看；`TOOL_FUNCTIONS` 给后端执行用。
- `tool call` 不是最终答案，`tool output` 也不是最终答案；最终答案是模型读取 tool output 后生成的自然语言回答。
- `Safety` 应写入 Agent 一句话理解，因为只要 Agent 能调用工具，就必须有后端安全边界。
- `audit` 是安全留痕，适合高风险工具调用；第 25 章只需提到，第 26 章知道它属于后端执行链路，后续工程化再展开。
