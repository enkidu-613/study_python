5. **Pydantic BaseModel vs SQLAlchemy Base 概念辨析（ADHD友好教学）**
   - **Pydantic BaseModel** = 快递单/服务员：校验API进出的JSON数据格式，在内存里工作，不管数据库
   - **SQLAlchemy Base** = 仓库货架图纸/仓库管理员：定义数据库表结构，负责CRUD和SQL生成，只和数据库打交道
   - 完整请求流程：JSON进来 → Pydantic校验 → 转ORM对象存数据库 → 查询返回ORM → 自动转JSON响应
   - 核心区别表对比（类比/职责/工作位置/字段定义/出错表现/对应文件）
   - 常见坑：不能直接把Pydantic对象传给db.add()，需要取字段新建ORM对象
   - 提到了 `from_attributes = True`（原orm_mode）配置可直接返回ORM对象
