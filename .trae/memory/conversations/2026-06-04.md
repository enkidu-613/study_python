**双存储流程**:
```
DocumentIn → SQLite 存完整文档 → TextSplitter 切片 → for 循环:
  ├─ SQLite 存 DocumentChunk 元数据
  └─ 拼 LangChain Document 对象 (lc_docs + chroma_ids)
→ db.commit() → vectorstore.add_documents(lc_docs, ids=chroma_ids)
```

### 2. RecursiveCharacterTextSplitter 切片原理

**配置**:
```python
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=80, chunk_overlap=10,
    separators=["\n\n", "\n", "。", "！", "？", "，", " ", ""],
)
```

**递归切分逻辑**: 按分隔符优先级逐级降级尝试切分点
```
"\n\n" → "\n" → "。" → "！" → "？" → "，" → " " → ""（硬切兜底）
```
在 chunk_size 附近找最语义化的断点，相比手搓版固定 80 字符硬切更智能。

**重叠**: `chunk_overlap=10`，相邻片段重叠 10 字符，防止关键信息落在边界被割裂。

### 3. truncate_context 上下文截断

**作用**: 格式化后的检索结果按块累积，Token 超预算即停止（贪心截断，因为 ChromaDB 已按相似度排序）。

**与 RunnableLambda 的误解**:

- 已清理误导性注释，移除未使用的 `RunnablePassthrough` 导入

### 4. 双存储版本对比（手搓版 vs LangChain 版）

| 步骤 | 手搓版 | LangChain 版 |
|------|--------|-------------|
| 切片 | 自己写 while 硬切 | 框架组件，语义感知切分 |
| Embedding | 循环里手动调 API，拿浮点列表 | 最后一步 `add_documents()` 自动批处理 |
| 给 ChromaDB 传参 | 4 个数组（ids, embeddings, documents, metadatas） | 传 Document 对象 + ids，embedding 内部算 |
| metadata | 3 字段 | 4 字段（多了 source） |
| Chroma 写入 | 底层 `collection.add()` | 高层 `vectorstore.add_documents()` |

**数组对齐**: 两版都靠同一次循环内 Append 保证下标对齐。LangChain 版从 4 数组减为 2 数组（`lc_docs` + `chroma_ids`），因为 `LCDocument` 打包了文档+元数据，Embedding 推迟到 `add_documents` 内部。

### 5. chroma_ids 的作用

`chroma_ids.append(f"doc{document.id}_chunk{i}")` 在循环中构建，最终传给 `vectorstore.add_documents(lc_docs, ids=chroma_ids)`。作用：给 ChromaDB 每条向量指定唯一主键，防止重复插入，方便按 ID 定位。

### 6. 管道 vs 中间件

LCEL 管道是**单向数据流**（数据A → 数据B → 数据C → 数据D），不是中间件。中间件是环绕式（请求进 → 层层包装 → 回传响应），管道是流水线式（每步加工后交给下家）。

### 7. SQL + 向量配合的实际场景

- 元数据预过滤（最常见）：先 SQL 筛选，再在缩小范围内做语义检索
- 结构化数据二次检索：向量检索后 SQL 查关联表增强回答
- 混合检索：向量 + 关键词各跑一遍，RRF 合并

当前项目的 SQLite 回查是学习实验性质，生产环境 metadata 完整时不需要。

### 3. FastAPI lifespan 预加载 Embedding 模型

**问题**: 首次调用 Embedding 耗时 17 秒（加载模型到内存），HTTP 请求超时无响应，表现像是"卡住了"。

**解决**: `main.py` 加入 lifespan 启动钩子，服务器启动时先加载模型再接受请求：
```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    from local_embedding import get_embedding_model
    get_embedding_model()  # 启动时预加载 ~10-20s
    yield
```
