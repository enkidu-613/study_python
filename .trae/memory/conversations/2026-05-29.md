## 对话 22: DocumentIn / DocumentChunk 模型问答

**时间**: 2026-05-29
**用户问题**: 
1. `routers/rag_router.py` 中的 `DocumentIn` 模型是完整文档的吗？
2. 有没有切片的模型？

**核心教学内容**:

### DocumentIn = 完整文档
- `DocumentIn` 是 Pydantic 模型，对应用户 POST 的完整文档
- 用户传入完整内容 → 代码内部 `split_text()` 切片 → 双存储
- `Document` (ORM) = SQLite `documents` 表，存完整文档

### 切片模型 = DocumentChunk (ORM)
- `DocumentChunk` = SQLite `document_chunks` 表，ORM 模型，定义在 `models.py`
- 字段：`id`, `document_id`（外键）, `chunk_index`, `content`（80字碎片）, `embedding_id`（绳子①）
- 没有单独的 Pydantic 模型，因为切片是代码内部自动生成的，不是用户传的
- 以后如果需要返回切片列表，可以补一个 `ChunkOut(BaseModel)`

### 数据流向
```
用户 POST DocumentIn（完整）
  → split_text() 切片
  → SQLite: Document（完整）+ DocumentChunk（碎片）
  → ChromaDB: 碎片向量
```

## 对话 24: Python async 与 JavaScript async 深度对比

**时间**: 2026-05-29
**用户问题**: 
1. Python async 函数和 JS 的有什么区别？
2. `asyncio.get_event_loop()` 显式获取是什么意思？
3. Promise 等价的 asyncio.Future / Task 的使用方法详细对比

**核心教学内容**:

### Python vs JS async 核心区别
- **Event Loop 归属**: Python 显式管理（`asyncio.run()`），JS 隐式自动（浏览器内置）
- **顶层 await**: Python ❌ 不行，JS ✅ 可以
- **返回对象**: Python 返回协程对象（coroutine，未执行），JS 返回 Promise（已开始执行）
- **触发执行**: Python 需要 `await` / `create_task` / `asyncio.run`，JS 自动执行

### asyncio.get_event_loop() 详解
- **作用**: 获取当前线程的 Event Loop，没有就创建
- **三种方式**: `get_event_loop()`（通用）、`get_running_loop()`（协程内部）、`new_event_loop()`（高级/多线程）
- **生命周期**: 获取 → 运行 → 关闭
- **日常代码**: 用 `asyncio.run()` 或框架（Uvicorn），不手动管 loop

### Promise vs asyncio.Task/Future 详细对比
| JS | Python | 说明 |
|----|--------|------|
| `Promise` | `asyncio.Task` | 最常用，创建即执行 |
| `new Promise((resolve, reject) => {})` | `asyncio.Future` | 底层占位符，手动控制完成 |
| `Promise.all([a, b])` | `asyncio.gather(a(), b())` | 并发全部 |
| `Promise.race([a, b])` | `asyncio.wait(..., FIRST_COMPLETED)` | 竞速 |

### 关键结论
- Python 和 JS 的 `async/await` 语法几乎一样，核心区别在 Event Loop 管理
- Python 协程调用后需要显式 await 才能执行，JS Promise 创建即执行
- 写业务代码时（FastAPI），框架帮你管好 loop，直接 await 就行
