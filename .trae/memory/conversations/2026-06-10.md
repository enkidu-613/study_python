## 对话 49: Python asyncio 三种协程对象与并发模式深入

**时间**: 2026-06-10
**用户任务**:
1. 深入理解 Future 回调机制与状态控制
2. 探索让 Python async 更像 JS Promise 的库
3. 学习三种并发模式及 Semaphore 限流

**核心教学内容**:

### 三种协程对象
- **Coroutine**：`async def` 返回的"待办清单"，未执行，遇到 `await` 才跑
- **Task**：`create_task` 包装的"已排队任务"，创建后立即在后台跑，可 cancel
- **Future**：底层占位符，用于桥接回调式代码，`set_result` / `set_exception` 控制返回状态

### Future 回调机制
- 回调函数内可根据数据状态决定调用 `set_result()` / `set_exception()` / `cancel()`
- 从而控制 `await future` 的返回结果或异常
- Future 只能设置一次，设置后不可更改

### Python async vs JS Promise
- JS 的 `.then().catch()` 链式调用在 Python 中可用 `returns` 库或自封装 `Promise` 类模拟
- `httpx` 的 API 最接近 JS 的 `fetch`
- Python 原生无 `.then()`， Guido 推崇显式优于隐式

### 三种并发模式
- **`gather`**：同时启动，全部完成后按传入顺序返回（等齐上菜）
- **`create_task`**：先下单后取餐，`await` 顺序任意，任务已在后台跑
- **`as_completed`**：谁先好先处理，返回顺序不固定（抢头啖汤）

### Semaphore 并发限制
- `asyncio.Semaphore(n)` 发 n 张通行证
- `async with semaphore:` 拿号进场，满了自动等待，出块自动还号
- 10 个 URL 限流 3 个并发 → 3 个 3 个执行，全部都会完成，不会丢弃任务

### Python 解包（Unpacking）
- `a, b = [1, 2]` 对应 JS 解构赋值
- `a, *rest = [1, 2, 3, 4]` 剩余解包，`*` 可在任意位置
- `**dict` 在函数调用中展开关键字参数
