## 对话 77: Prompt Engineering 进阶结构化输出、Few-Shot、采样参数、环境变量、LangChain Chain 与异常处理

时间: 2026-06-26

### 完成事项

1. **结构化输出与 Schema 思想**
   - 明确 `Schema` 是跨语言通用的“声明式数据契约”思想。
   - 对比 Pydantic Schema 与 Prisma schema：前者约束 Python/API/LLM 输出，后者约束数据库结构和迁移。
   - 新增基础笔记：`md/编程语言基础/Schema_声明式数据契约.md`。

2. **第 21 章核心工作思想**
   - 明确本章不是普通 Prompt 大全，而是 Prompt Engineering 的工程化升级版。
   - 普通 Prompt 追求“模型答得好”；工程化 Prompt 追求“模型输出能被程序稳定消费”。
   - 核心链路：自然语言输入 → Prompt/Few-Shot → LLM → 结构化输出 → Pydantic 校验 → FastAPI 响应。

3. **Zero-Shot、Few-Shot 与任务提取器边界**
   - Few-Shot 不是训练模型，而是本次 Prompt 里的临时例题。
   - `human -> ai` 示例用于展示“例题 → 标准答案”，最后一个 `human` 才是真实输入。
   - 区分任务提取器与任务规划器：Extraction 问“这句话里有什么”，Planning 问“为了完成目标要做哪些步骤”。

4. **采样参数 temperature/top_p**
   - 澄清 `temperature` 和 `top_p` 不是直接控制“是否听话”或“关键词准确度”的参数。
   - 它们控制模型生成 token 时的随机性；抽取、分类、JSON 输出优先使用 `temperature=0`。
   - `top_p` 不显式传入时使用 provider/SDK 默认值；学习阶段先只调 `temperature`。

5. **第三方库、import 与环境变量**
   - 建立“库角色表”学习法：先认库的职责，再学最小调用模板。
   - import 不靠硬背库名，按对象来源查：教程示例、项目搜索、报错、官方文档。
   - 理解 `load_dotenv()` 与 `os.getenv()` 分工：`dotenv` 读取 `.env` 并写入 `os.environ`，`os.getenv()` 从环境变量表读取。

6. **LangChain Chain 与 `ainvoke()`**
   - `prompt | llm.with_structured_output(...)` 会构建 LangChain Runnable chain 对象。
   - `get_task_extractor()` 返回的是 chain；`ainvoke(input)` 是异步执行整条 chain，不只是传参。
   - `ainvoke({"text": value})` 的 key 必须匹配 Prompt 里的 `{text}` 占位符。
   - 推荐更清晰的写法：用 `{feedback_text}` 和 `prompt_input = {"feedback_text": feedback_text}` 提升可读性。

7. **惰性单例、日志与异常链**
   - `_task_extractor` 是惰性单例缓存：第一次构建，后续请求复用。
   - `logger = logging.getLogger(__name__)` 为当前模块获取带模块名的日志记录器。
   - `except Exception as exc` 中的 `exc` 是原始异常；`raise HTTPException(...) from exc` 保留异常因果链。
   - Python traceback 是错误调用链，调试时先看最后一行，再找项目文件行号。
