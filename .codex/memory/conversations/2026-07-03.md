### 关键结论

- `page_content/document` 是原文正文，`embedding vector` 是模型算出的向量，`metadata` 是过滤和回查标签。
- 当前项目 SQLite 保存业务文档、chunk 原文和 `embedding_id`；ChromaDB 保存向量、索引、document 与 metadata。
- RAG Evaluation 的核心是检查证据链，不只看最终回答是否像真的。
- `/langchain-rag/search` 用于看检索证据，`/langchain-rag/chat` 用于看最终回答。
- 对话历史解决“聊到哪了”，retrieval trace 解决“证据从哪来、rank/score 是多少”。
- `eval_cases` 当前阶段先作为人工检查清单：`case_id`、`question`、`expected_doc`、`expected_fact`。
- 上下文太脏时先看 `/search`，再考虑调小 `n_results`、检查 chunk、metadata 过滤、相似度阈值，进阶再 rerank。
- RAG 调参应一次只改一个变量，使用同一组 eval cases 做对比。
