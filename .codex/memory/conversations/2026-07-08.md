### 关键结论

- `create_agent` 可以看作第 26 章手写 loop 的框架版：它把模型、工具列表、工具执行循环和可选记忆组合成一个可调用对象。
- `@tool` 是把 Python 函数包装成 LangChain Tool 的装饰器，会读取函数名、类型注解和 docstring 来生成工具说明。
- 工具内部可以使用 `Document`，但给模型的 tool output 最好是短、清楚、可引用的字符串。
- `thread_id` 表示某一条对话线程，不等于 `user_id`；Agent 要有 `checkpointer + thread_id` 才能恢复同一段对话。
- 当前项目的 `registry.py` 继续保留为手写 Function Calling 学习代码；第 28 章用 `@tool + create_agent` 进入框架写法。
